# 2025年7月現在の Cursor と VS Code + Github Copilot の比較

**URL**: [2025年7月現在の Cursor と VS Code + Github Copilot の比較](https://zenn.dev/irsc/articles/2d33584ac7978a)  
**取得日**: 2025-11-07

## Original Content

AIコーディングアシスタントの2大ツールであるCursorとGitHub Copilotについて、2025年7月時点での最新アップデートを踏まえた実践的な比較記事。料金体系の刷新、新機能の追加、実際の使用感に基づいた選択基準を提示。

### 主要なアップデート（直近60日）

**Cursor側**
- v1.0→v1.2で Bugbot GA（$40/月）、Background Agent GA、Jupyter高速化
- 料金モデル刷新：500 fast req → 毎月$20クレジット＋Auto無制限

**Copilot側**
- 標準モデルがGPT-4.1に更新
- Agent Mode / MCP が全ユーザGA
- Pro プランで Premium 300 req の無料枠確定

### 料金比較（Claude 4 Sonnet使用時）

| 月間リクエスト数 | Copilot Pro | Cursor Pro |
|----------------|-------------|------------|
| 100回 | $10（枠内） | $20（枠内） |
| 500回 | $28 | $29 |
| 2,000回 | $78 | $36 |
| 5,000回 | $198 | $90 |

**指針**：
- ≤300 req → Copilot は追加0円
- 軽量＋大量 → Cursor が割安
- 長文 or 少量 → Copilot の $0.04/req が安定

### 機能面での差異

**Cursor の強み**
- 複数AIモデルの切替自由（GPT-4o / Claude3-4 / Gemini / Grok）
- Background Agent による大規模リファクタリング
- Bugbot によるPRバグ検知（+$40/月）
- 複数行別箇所コードの同時参照が便利

**Copilot の強み**
- 日本語チャットの安定性
- DevOps連携（GitHub Actions生成・PR要約）
- 軽量動作（VS Code拡張として）
- 定額$10＋Premium 300 reqの読みやすい課金

### 実装への影響

**データ分析・品質保証業務での選択基準**

1. **短いコード断片の頻繁な生成**（データ処理スクリプト、バリデーションコード）
   → Cursor Pro のクレジット制が有利

2. **日本語ドキュメント・コメント重視**
   → Copilot の日本語UXが安定

3. **大規模リファクタリング**（レガシーコード改善、アーキテクチャ変更）
   → Cursor の Background Agent が強力

4. **CI/CD統合重視**（テスト自動化、品質ゲート）
   → Copilot の GitHub Actions連携

## Summary

1. **Cursorは2025年7月に料金体系を刷新**し、従来のリクエスト数制から「$20/月クレジット＋超過従量」方式に変更。短いリクエストを多用するワークフローに有利な課金モデルとなった

2. **GitHub Copilotは標準モデルがGPT-4.1に更新**され、Pro プラン（$10/月）でもAgent ModeとPremium 300リクエストの無料枠が利用可能に。コスト予測性が高く、少量利用時は追加課金なし

3. **Claude 4 Sonnetを月2,000回以上使う場合**、Cursorの方がコスト効率が良い（$36 vs $78）。軽量リクエストを大量に処理するデータ分析・品質保証業務では、Cursorのクレジット制が経済的

4. **Cursorの新機能 Bugbot（$40/月追加）**は、PRに対して論理バグやセキュリティバグを自動コメント。品質保証業務における自動レビュー層として価値が高いが、月額合計$60となるため費用対効果の検討が必要

5. **日本語での開発を重視する場合はCopilotが優位**。Cursorは英語優位で日本語チャットにバグ（コードブロック表示のズレ）が発生することがある。ドキュメント作成や日本語コメントが多い業務では要注意

## Additional Information: GitHub Agent HQ（2025年10月発表）

### Agent HQとは

GitHub Universe 2025（2025年10月28-29日）で発表された、**複数のAIコーディングエージェントを単一プラットフォームに統合する新構想**。「エージェント本部（Agent Headquarters）」として、GitHubが各社のAIエージェントを統合管理するハブとなる。

### 背景と目的

現状のAI開発環境は、各社が独自のエコシステムを構築し分断されている（「Git登場以前の混沌」に類似）。Agent HQはこれを解決し、開発者が単一のワークフローで複数のAIエージェントを効率的に活用できる環境を提供する。

### 主要機能

**1. ミッションコントロール（Mission Control）**
- GitHub.com、VS Code、CLI、モバイルアプリから統一的にエージェントを操作
- 複数エージェントのタスクを一元管理・監視
- エージェント作成コードのブランチコントロール
- Slack/Linear等との統合

**2. サードパーティAIエージェント統合**
今後数ヶ月以内に以下のエージェントが利用可能（予定）：
- Anthropic Claude Code
- OpenAI Codex（Copilot Pro+で先行利用開始）
- Google Gemini
- Cognition Devin
- xAI Grok

**重要：料金詳細は未発表**
- 公式発表では「既存の有料Copilotサブスクリプションの一部として」とのみ記載
- 現行プラン（Pro $10/月、Pro+ 約$60/月等）に含まれるのか、追加課金が必要かは不明
- 詳細は「今後数ヶ月以内」に発表予定

**3. セキュリティ・ガバナンス機能**
- エージェントコントロールプレーン（パブリックプレビュー）
- 監査ログによるエージェント動作の追跡
- GitHub Actions環境でのサンドボックス実行
- 指定ブランチへのコミット制限
- MCP（Model Context Protocol）レジストリの統合管理

**4. コード品質管理機能**
- GitHub Code Quality：保守性・信頼性への影響を自動分析
- Copilot usage metrics dashboard：組織内AI利用状況の可視化
- エージェントによるコードレビュー自動化

### Agent HQの戦略的意味

**「統合戦争」の転換点**
- 従来：各社が独自エコシステムで囲い込み
- Agent HQ：「AIエージェントのApp Store」として開放的に運営
- 開発者の選択自由度が大幅向上
- AIプロバイダー間の競争激化

**品質保証業務への影響予測**

1. **複数エージェントの使い分けが可能に**
   - データ分析：特化エージェントA
   - バグ検出：Bugbot（Cursor）+ GitHub Code Quality
   - レビュー：Claude Code（自然言語理解）
   - テスト生成：Codex（コード生成特化）

2. **統合管理による効率化**

   - ミッションコントロールで進捗一元管理
   - ツール切り替えコストの削減

3. **ガバナンス強化**
   - 監査ログによる品質保証プロセスの可視化
   - エージェント動作の標準化（AGENTS.md）
