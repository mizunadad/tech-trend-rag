# Claude Codeと自身の脳を繋ぐ、ナレッジ管理CLIツールによるMCPサーバー接続

**🔗 URL**: [Claude Codeと自身の脳を繋ぐ、ナレッジ管理CLIツールによるMCPサーバー接続](https://zenn.dev/boykush/articles/1aa8848b23f09a)  
**📅 Date**: 2025-09-15

## 📄 Original Content

Boykush氏による、独自開発のナレッジ管理CLIツール「Scraps」にMCPサーバー機能を実装し、Claude Codeと連携させた実践記事。Wikiリンク記法で書いたマークダウンファイルを静的サイトへビルドするCLIツールに、Model Context Protocol（MCP）サーバー機能を追加することで、個人のナレッジベースをClaude Codeから直接アクセス・活用可能にした。従来のObsidianプラグイン方式とは異なり、マークダウンファイルさえあれば軽量に開始できる点が特徴。実際の使用例として、BDD（Behavior Driven Development）関連情報の検索・整理、個人ナレッジを反映したエージェントの作成などを紹介し、LLMに自身の「脳」を拡張として統合する新しいアプローチを実証している。

## 🎯 Summary

- **軽量ナレッジ連携の実現**: CLIベースのScrapsツールにMCPサーバー機能を追加し、マークダウンファイルのみでClaude Codeとの連携を可能にする軽量なナレッジ管理システムを構築
- **5つの専用MCPツール群**: タイトル検索、内部リンク取得、バックリンク確認、タグ管理、タグバックリンク検索により、個人ナレッジベースの多角的アクセスが可能
- **Web検索不要の高精度情報取得**: BDD導入方法の問い合わせに対し、個人ナレッジベースのみから実例マッピング、Gherkin記法、Screenplay Patternなど精度の高い専門情報を即座に提供
- **パーソナルエージェントの自動生成**: 個人ナレッジを反映したサブエージェント作成により、まるで「もう1人の自分」との対話を実現し、次の学習分野（圏論、エフェクトシステム等）を的確に提案
- **LLMへの脳拡張統合**: 従来の汎用LLMではなく、個人の蓄積知識を直接活用することで「オレが3人分になる」状態を実現し、個人特化型AI アシスタントの新形態を提示

## 📊 Scraps MCPツール一覧

| ツール名 | 機能 | 用途 | 出力内容 |
|---|---|---|---|
| search_scraps | タイトル曖昧検索 | キーワードベースでの記事発見 | マークダウンコンテンツ含む |
| lookup_scrap_links | 内部リンク一覧 | 特定記事からの参照先確認 | リンク先記事一覧 |
| lookup_scrap_backlinks | バックリンク一覧 | 特定記事への参照元確認 | 参照元記事一覧 |
| list_tags | タグ一覧取得 | 分類体系の把握 | 全タグリスト |
| lookup_tag_backlinks | タグバックリンク | タグ関連記事の収集 | タグ付き記事一覧 |

## 📊 従来手法との比較

| 項目 | Obsidian + MCP | Scraps + MCP | 汎用LLM |
|---|---|---|---|
| 導入複雑さ | 中（プラグイン設定） | 低（CLIのみ） | 低 |
| 軽量性 | 中 | 高 | 高 |
| 個人特化度 | 高 | 高 | 低 |
| 情報精度 | 高（ナレッジベース） | 高（ナレッジベース） | 中（汎用知識） |
| Web検索依存 | なし | なし | あり |
| 学習提案精度 | 高 | 高 | 中 |

## 💭 My Notes

- ナレッジ管理とAI連携の新しいアプローチとして非常に興味深い
- 品質保証の観点から、個人ナレッジの信頼性担保メカニズムが重要
- 製品設計において、同様の手法で設計ドキュメントやテスト知見の活用が可能
- 子供たちの学習においても、個人学習記録との連携でパーソナライズド学習支援が実現できそう
- データ解析業務での過去分析結果やPythonコード資産との連携に応用価値あり
