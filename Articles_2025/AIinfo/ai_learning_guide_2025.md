## 🎯 LLMの特性を活かすプロンプト戦略（ユースケース別）

LLMは「パターンマッチング」に優れ、既知領域に強い特性を理解した効果的なプロンプト設計：

### 📊 分析・調査タスク
**特性**: 既知パターンの組み合わせが得意、データの構造化に優秀

```
【推奨プロンプト型】
「以下のデータを分析し、[既知の分析フレームワーク名]に基づいて整理してください：
1. データの概要と特徴
2. [具体的な分析軸]での分類
3. パターンや傾向の抽出
4. 既知の類似事例との比較
5. 分析結果の信頼性レベル（高/中/低）を明記」
```

### 🔍 品質チェック・検証タスク  
**特性**: チェックリスト処理、既知基準との照合が正確

```
【推奨プロンプト型】
「以下を[具体的な品質基準名]に照らして検証してください：
- 検証対象：[具体的な対象]
- 適用基準：[明確な基準リスト]
- 評価方法：各項目を○/△/×で評価
- 不具合発見時：類似の既知事例と対策を提示
- 検証の限界：AIでは判断困難な領域を明記」
```

### 📝 文書作成・要約タスク
**特性**: 構造化された形式への変換、既存形式の模倣が得意

```
【推奨プロンプト型】
「以下の情報を[具体的な文書形式]で作成してください：
- 対象読者：[明確な読者設定]  
- 参考形式：[既存の良い事例を提示]
- 必須要素：[含めるべき項目リスト]
- 文体・トーン：[具体的な指定]
- 検証ポイント：重要事実は元情報の引用を併記」
```

### 🛠️ 問題解決・トラブルシューティング
**特性**: 既知の解決パターンの適用、段階的アプローチが有効

```
【推奨プロンプト型】
「以下の問題を既知の解決手法を用いて段階的に分析してください：
1. 問題の分類：[類似の既知問題カテゴリ]
2. 原因仮説：一般的な原因パターンから3つ選定
3. 検証手順：各仮説の確認方法
4. 解決策：既知の対策から適用可能性順に提示
5. 注意事項：このアプローチで対応困難な要素を明記」
```

### 📚 学習・教育サポート
**特性**: 段階的説明、既知の教育パターンの活用が効果的

```
【推奨プロンプト型】
「[学習対象]を[学習者レベル]向けに、以下の構造で説明してください：
- 前提知識：必要な基礎概念を確認
- 段階的説明：既知の概念から新概念への橋渡し
- 具体例：実際の[業界/分野]での活用事例
- 練習問題：理解度確認用の段階別問題
- 参考資料：更なる学習のための信頼できるソース」
```

### 💻 コーディング・技術タスク
**特性**: 既存パターンの組み合わせ、ベストプラクティスの適用

```
【推奨プロンプト型】
「以下の要件で[言語名]コードを作成してください：
- 機能要件：[具体的な機能記述]
- 制約条件：[環境・性能制約]
- 参考パターン：[既知のデザインパターン名]
- 品質要件：テスト可能性、可読性、保守性を重視
- 出力形式：コード + 設計思想の解説 + テストケース例」
```

### ⚠️ プロンプト設計時の重要原則

| 原則 | 理由 | 実装方法 |
|---|---|---|
| **既知パターンの活用** | LLMのパターンマッチング能力を最大化 | 具体的なフレームワーク・手法名を明記 |
| **段階的指示** | 複雑タスクの分解で精度向上 | 番号付きステップで構造化 |
| **検証可能性の確保** | AI判断の限界を明確化 | 信頼性レベル・不確実性の明記を要求 |
| **具体性の重視** | 曖昧な指示による品質劣化を防止 | 抽象的表現を避け、具体例を併記 |
| **フィードバックループ** | 出力品質の継続改善 | 結果評価と次回改善点を含む設計 |

## 💭 My Notes

- 品質保証の観点から、LLMの「パターンマッチング」特性は既知の品質基準には高い再現性を示すが、新規課題での予測困難性が重要な制約
- 電力制約は半導体産業にも影響大：データセンター向け高性能チップの需要増と電力効率改善の技術的トレードオフが課題
- 2028年データ枯渇問題は、合成データ品質検証技術の重要性を示唆：人工データの品質指標開発が急務
- CoT推論の限界理解は、AI活用業務での適用範囲設定に重要：既知パターンの品質チェックには有効、創造的問題解決には限界
- エネルギー効率の観点から、エッジAI・小型モデルへの回帰トレンドが品質保証分野での実装可能性を高める
