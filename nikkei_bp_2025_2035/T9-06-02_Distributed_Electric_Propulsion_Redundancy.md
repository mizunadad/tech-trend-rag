# T9-06-02 分散電動推進システム・冗長設計

## Summary（5つの要点）

1. **技術概要**: 複数の小型電動モーターを機体各所に配置し、1つまたは複数が故障しても飛行を継続できる冗長設計。従来ヘリコプターの「単一エンジン故障=墜落」リスクを根本的に解消し、eVTOL独自の技術的優位性を実現

2. **システム構成**: バッテリーパック、モーター、インバーター、制御系の各要素を二重化・三重化。6-18個のモーター・ローターを配置し、任意の2-3個が停止しても安全飛行可能。電力系統も複数系統に分離し、単一故障点（Single Point of Failure）を排除

3. **安全性向上効果**: 航空機安全基準で要求される10⁻⁹（10億時間に1回）の壊滅的故障率を達成。商用航空機（ボーイング、エアバス）と同等の安全レベルをeVTOLで実現し、都市上空飛行の社会受容性を確保

4. **制御技術**: 故障検知→診断→再構成を数ミリ秒で実行するFault Tolerant Control（FTC: 故障許容制御）。残存モーターの出力再配分、飛行モード変更（高速→安定飛行）、緊急着陸地点探索をAIが自動実行

5. **認証要求**: FAA、EASA、国土交通省航空局が冗長設計を必須要件化。各系統の独立性証明、共通原因故障（Common Mode Failure）排除、ソフトウェア信頼性（DO-178C Level A）が認証取得の鍵

## 具体的プロダクト事例

### 日本企業の先進事例

**SkyDrive SD-05 冗長システム（日本）**
- 企業: SkyDrive株式会社
- 構成: 8個の独立モーター・ローター、4系統の独立電力系統
- 冗長性: 任意の2つのモーターが故障しても安全着陸可能（2 Out of 8 Failure Tolerance）
- 制御: 各モーターに専用インバーター配置、故障検知は10ms以内
- バッテリー: 2系統の独立バッテリーパック、片系統故障でも15分飛行可能
- 特徴: 国土交通省と協議し、日本初の有人eVTOL認証基準策定に貢献
- リンク: https://skydrive2020.com/

**川崎重工 K-RACER X2 無人機実証（開発中）**
- 企業: 川崎重工業（防衛・航空宇宙部門）
- 用途: 有人eVTOL技術実証のため、まず無人機で冗長システム検証
- 特徴: ティルトローター型、軍用ヘリコプター開発の信頼性技術を民生転用
- 冗長性: 油圧系統の二重化経験を電動システムに応用
- リンク: https://www.khi.co.jp/

### グローバルスタンダード

**Joby Aviation 分散電動推進システム（米国）**
- 企業: Joby Aviation
- 構成: 6個のティルトローター（前方4個、後方2個）、各ローターに独立モーター
- 冗長性: 任意の1つのモーター故障でも全飛行プロファイルで安全飛行継続可能
- 電力系統: 3系統の独立高電圧バス（各バスに複数バッテリーパック接続）
- 制御アーキテクチャ: トリプルモジュール冗長（TMR: Triple Modular Redundancy）飛行制御コンピューター
- 認証対応: FAA Part 23認証要求を満たす冗長設計、DO-178C/DO-254準拠
- リンク: https://www.jobyaviation.com/

**Volocopter 18ローター冗長設計（ドイツ）**
- 企業: Volocopter
- 構成: 18個の独立ローター・モーター（最多クラス）
- 冗長性: 任意の4個のモーターが故障しても飛行継続可能（極めて高い冗長性）
- 設計哲学: 「Inherent Safety（本質安全）」- 多数の小型モーターで故障影響を最小化
- バッテリー: 9個の独立バッテリーパック、各2つのモーターに電力供給
- 実証: 2019年シンガポール飛行で複数モーター停止テスト成功
- リンク: https://www.volocopter.com/

**Lilium Jet ベクタード・スラスト冗長性（ドイツ）**
- 企業: Lilium
- 構成: 36個の電動ジェットエンジン（前翼12個、主翼24個）
- 冗長性: 複数エンジン停止でも推力ベクトル制御で姿勢維持、安全着陸可能
- 特徴: 固定翼機の空力的安定性＋分散推進の冗長性を両立
- 電力系統: 10個の独立バッテリーパック
- 制御: ベクタード・スラスト（推力方向制御）により、エンジン故障時の姿勢制御を最適化
- リンク: https://lilium.com/

**Beta Technologies ALIA ティルトローター冗長設計（米国）**
- 企業: Beta Technologies（Amazon等が出資）
- 構成: 4個のティルトローター＋1個の後方プッシャープロペラ
- 冗長性: 固定翼飛行時はプロペラのみで飛行可能、ローター故障でもグライダーとして着陸
- 用途: 貨物輸送重視、航続距離250マイル（400km）
- 認証: FAA Part 23認証取得プロセス進行中
- リンク: https://www.beta.team/

## 技術解説詳細

### ハイプ・サイクル上の位置づけ

**現在位置**: 啓蒙活動期（実証から実用化へ）

- **黎明期（2010-2015）**: ドローンの多ローター技術を有人機に応用する構想
- **過度な期待のピーク（2016-2019）**: 「eVTOLはヘリコプターより安全」という主張が広がる
- **幻滅期の谷（2020-2022）**: 認証当局が厳格な安全基準を要求、技術的ハードル明確化
- **啓蒙活動期（2023-2026）**: 飛行実証で冗長性を実証、認証取得プロセスで安全性を証明
- **生産性の安定期（2027-2035）**: 認証済みeVTOLが商用運航、冗長設計が業界標準に

### 冗長設計の技術階層

**レベル1: ハードウェア冗長性**
- モーター・ローターの複数配置（6-18個）
- バッテリーパックの系統分離（2-10系統）
- インバーター（モータードライバ）の独立化
- 電力バス（配電線）の物理的分離

**レベル2: 電力系統冗長性**
- 高電圧DC配電システムの多重化（典型的には3系統）
- バッテリーセルの並列・直列構成最適化
- DC/DCコンバーター冗長化
- 絶縁監視システム（各系統の地絡検知）

**レベル3: 制御系冗長性**
- 飛行制御コンピューター（FCC: Flight Control Computer）のトリプルモジュール冗長
- センサー冗長化（IMU、GPS、気圧計を各3個以上）
- 通信バス冗長化（CAN、ARINC 429、Ethernet等）
- アクチュエーター（サーボ、モーター制御）の二重化

**レベル4: ソフトウェア冗長性**
- 異なる開発チームが独立にソフトウェアを開発（Dissimilar Software）
- 投票機構（Voting Mechanism）で多数決判定
- 故障検知・診断・分離・再構成（FDIR: Fault Detection, Isolation, and Reconfiguration）
- セーフモード自動移行（緊急着陸モード、最小機能維持モード）

### 品質保証・信頼性の視点

**FMEA/FTA手法の徹底適用**
- **FMEA（故障モード影響解析）**: 全部品の故障モードとその影響を網羅的に分析
  - 例: モーターA故障→残り5モーターで出力再配分→安全飛行継続
- **FTA（故障の木解析）**: 墜落に至るイベントツリーを逆算し、単一故障点を排除
  - 目標: 10⁻⁹ /flight hour（10億時間に1回の壊滅的故障率）

**Common Mode Failure（共通原因故障）の排除**
- 同一ロットのバッテリーセル使用を避ける（異なるサプライヤーから調達）
- ソフトウェアのバグが全系統に影響しないよう、異なるアルゴリズム実装
- 電磁干渉（EMI）による同時故障防止（シールド、配線分離）
- サイバー攻撃対策（暗号化通信、侵入検知システム）

**メカトロニクス品質保証への示唆**
- 半導体部品の耐久性試験（温度サイクル、振動、衝撃）
- バッテリーマネジメントシステム（BMS）のセル単位監視・均等化
- モーター制御のトルクリップル最小化（振動・騒音低減）
- インバーターのスイッチング損失低減（SiC/GaN半導体活用）

### データ解析・Python活用領域

1. **故障シミュレーション**: モンテカルロシミュレーションで様々な故障パターンの影響評価
2. **飛行データ解析**: 飛行試験データから故障予兆を機械学習で検出
3. **最適制御アルゴリズム**: Python（SciPy、CVXPY）で最適化問題を解き、故障時の出力再配分を計算
4. **信頼性予測**: ワイブル分布等で部品寿命予測、予防保全計画策定

```python
# 例: 故障時の推力再配分最適化（簡略版）
import numpy as np
from scipy.optimize import minimize

# 8モーター構成、モーター3が故障
num_motors = 8
failed_motor = 3
target_thrust = 1000  # N

def objective(thrusts):
    # 各モーターの推力差を最小化（均等配分）
    return np.var(thrusts)

def constraint_total_thrust(thrusts):
    return np.sum(thrusts) - target_thrust

# 最適化実行
x0 = np.ones(num_motors) * target_thrust / (num_motors - 1)
x0[failed_motor] = 0
bounds = [(0, 200) if i != failed_motor else (0, 0) for i in range(num_motors)]
constraints = {'type': 'eq', 'fun': constraint_total_thrust}
result = minimize(objective, x0, bounds=bounds, constraints=constraints)

print("最適推力配分:", result.x)
```

## 日本の立ち位置・競争力分析（4つの要点）

### 1. 強み：産業用ロボット・制御技術の蓄積

- **冗長制御ノウハウ**: ファナック、安川電機等の産業用ロボットで冗長系設計の経験あり（双腕ロボット、協働ロボット）
- **モーター制御技術**: 日本電産（Nidec）、三菱電機のベクトル制御、センサレス制御技術は世界トップレベル
- **パワーエレクトロニクス**: 三菱電機、富士電機、東芝のSiC/GaN半導体技術は高効率インバーターに貢献
- **BMS技術**: パナソニック、GSユアサのバッテリーマネジメントシステムは電気自動車で実績あり

### 2. 強み：品質管理文化と信頼性工学

- **徹底したFMEA/FTA**: トヨタ生産方式、TQM（Total Quality Management）の文化が冗長設計に適合
- **長期信頼性試験**: 自動車部品で培った加速寿命試験（HALT）、信頼性試験手法を航空機に応用可能
- **サプライヤー管理**: ティア1→ティア2→ティア3のサプライチェーン品質管理が厳格
- **安全文化**: 鉄道、原子力発電等で培った「絶対安全」志向が航空機開発に適合

### 3. 弱み：航空機認証プロセスの経験不足

- **DO-178C/DO-254未経験**: 自動車のISO 26262経験者は多いが、航空機ソフトウェア・ハードウェア認証は別物
- **FAA/EASA対応**: 米国・欧州の認証当局との交渉経験不足、SkyDriveも手探り状態
- **試験設備**: 航空機用の環境試験設備（高度試験、着氷試験等）が日本国内に少ない
- **人材不足**: 航空機認証エンジニアが圧倒的に不足、海外人材獲得競争で不利

### 4. 弱み：システムインテグレーション経験

- **機体統合**: ボーイング、エアバス、ロッキード・マーチン等の大手航空機メーカーが無く、システム全体最適化の経験不足
- **サプライチェーン**: 航空機グレード部品（RTCA DO-160環境試験適合品）のサプライヤーが日本に少ない
- **シミュレーション環境**: モデルベース開発（MBSE）、Hardware-in-the-Loop（HIL）試験環境の整備が遅れ
- **国際標準化**: SAE、RTCA等の国際標準化団体への日本企業の参画が少ない

## 実装上の課題と解決策

### 課題1: 重量増加と効率低下

**問題**: 冗長系追加でバッテリー、モーター、配線が増加し、機体重量が増える
**解決策**:
- 軽量材料（CFRP、アルミリチウム合金）採用
- モーター小型化（高出力密度化）、SiC/GaN半導体で電力損失削減
- 最適化設計（トポロジー最適化）で必要最小限の冗長性を確保

### 課題2: 制御系の複雑化

**問題**: 故障検知・診断・再構成ロジックが複雑化し、ソフトウェアバグリスク増大
**解決策**:
- モデルベース開発（MATLAB/Simulink）でシミュレーション徹底
- 形式手法（Formal Methods）でソフトウェア検証
- HIL（Hardware-in-the-Loop）試験で実機相当の環境試験

### 課題3: コスト増加

**問題**: 冗長系により部品点数が増加し、製造コスト・保守コストが上昇
**解決策**:
- 量産効果（年産数百機以上）でコスト削減
- 自動車産業のサプライチェーン活用（automotive-grade部品の流用）
- 予知保全により、過剰な保守を削減
