# T9-02-04 デジタルツイン交通シミュレーション

## ハイプ・サイクル位置づけ
**現在フェーズ**: 幻滅期→啓発期（2025年）
- **過度な期待のピーク期** (2020-2022): PLATEAU開始、メタバース・デジタルツインブーム
- **幻滅期** (2023-2024): 費用対効果への疑問、ユースケース不足の顕在化
- **啓発期** (2025-): 実用的なユースケース確立、社会実装フェーズ開始
- **生産性の安定期** (2028-2030): 標準インフラとして定着予測

## 5つの要点サマリー

### 1. デジタルツイン交通シミュレーションとは
都市全体の交通システムをサイバー空間に高精度で再現し、リアルタイムデータと統合して渋滞予測、信号制御最適化、新路線の効果検証、MaaS設計をシミュレーションする技術。国土交通省のPLATEAU（プラトー）プロジェクトが3D都市モデルのオープンデータ化を推進し、2025年時点で約130都市の3D都市モデルが公開され、交通管理、防災、都市計画など多様な分野での活用が進展している。

### 2. 技術的構成要素
- **3D都市モデル**: CityGML 2.0国際標準形式で建物、道路、地形を数cm精度で3Dデジタル化。建物用途、階数、構造などの属性情報を付与
- **リアルタイムデータ統合**: 交通流データ、気象情報、イベント情報、公共交通運行状況をIoTセンサーとAPI連携で収集
- **点群データ活用**: LiDAR測量による高精度3D空間情報（静岡県、東京都が独自整備）
- **シミュレーションエンジン**: 交通流シミュレーション、人流解析、環境影響評価（騒音、大気質）
- **可視化技術**: WebGIS（Cesium、Deck.gl、Three.js）、ゲームエンジン（Unity、Unreal Engine）活用

### 3. 具体的な応用事例
- **交通事故リスク評価**: 三井住友海上が死角パラメータを開発し、愛媛県松山市で全公道・全交差点の事故リスクを可視化、EBPM（証拠に基づく政策立案）を実現
- **工事車両経路最適化**: 大阪市の大規模再開発で、住民生活圏や騒音レベルを考慮した搬入経路をシミュレーション
- **AGV自律走行**: PLATEAU 3D都市モデルから仮想LiDAR点群マップを生成し、無人搬送車の自己位置測位に活用
- **仙台市防災教育**: 東北工業大学が3D都市モデルを活用した小中学生向け災害リスク学習ツールを開発

### 4. 社会実装の課題と対策
- **費用対効果の理解促進**: 初期整備コストに対する長期的メリット（事故削減、渋滞緩和、防災効果）の定量評価が必要
- **自治体のノウハウ不足**: 産官学連携による人材育成、東京大学がPLATEAU活用の演習プログラムを提供
- **データ更新の持続性**: 初期整備後の継続的なデータ更新体制の構築、補助金依存からの脱却
- **3D都市モデルの精度限界**: 建物密集エリアでは高精度だが、郊外・未整備エリアでは精度不足の課題

### 5. 日本の競争力分析
**強み**: 国土交通省主導のPLATEAUによる全国統一基盤整備、オープンデータ化による利活用促進、自動運転・ドローン運航管理との統合研究の先行性
**弱み**: 初期導入コストの高さ、実用的ユースケースの不足、データ更新体制の脆弱性、海外（韓国MORAI自動運転シミュレーション、シンガポールVirtual Singapore）との技術競争

## 全体要約の特徴（5つの要点）

### 1. PLATEAU主導の国家プロジェクトとして推進
国土交通省が2020年度に開始したPLATEAUプロジェクトは、日本全国の3D都市モデルをCityGML 2.0国際標準形式で整備し、オープンデータ化する取り組みである。2025年時点で約130都市のデータが公開され、交通管理、防災、都市計画など多様な分野で活用が進展。補助金支援により自治体の導入が促進され、2025年度は本格的な社会実装フェーズに移行している。

### 2. 交通事故削減・渋滞緩和の具体的成果
三井住友海上とMS&ADインターリスク総研が開発した「死角パラメータ」は、3D都市モデルと組み合わせて交差点の事故発生リスクを高精度評価し、愛媛県松山市で全公道・全交差点の可視化を実現した。大阪市の大規模再開発では、工事車両の搬入経路を住民生活圏や騒音レベルを考慮してシミュレーションし、最適ルートを算出。これらはEBPM（証拠に基づく政策立案）の実践例である。

### 3. 自動運転・ドローン運航への応用展開
PLATEAU 3D都市モデルは自動運転車やドローンのシミュレーション環境として活用され、韓国MORAIのように実在都市での大規模実験が可能になる。AGV（無人搬送車両）の自律走行では、3D都市モデルから仮想LiDAR点群マップを生成し、事前の点群マップ取得コストを削減。ただし3D都市モデルで再現されていないエリアでは精度不足の課題が残る。

### 4. 費用対効果と持続性の課題
初期整備コストの高さ（1都市数千万円〜数億円）に対し、長期的な事故削減効果、渋滞緩和による経済効果、防災減災効果の定量評価が不十分であり、自治体の費用対効果理解を得にくい。補助金依存からの脱却、継続的なデータ更新体制の構築が課題。自治体職員のノウハウ不足に対しては、産官学連携による人材育成が進行中（東京大学がPLATEAU活用演習を提供）。

### 5. 国際競争と日本の差別化戦略
シンガポールのVirtual Singapore、韓国MORAIの自動運転シミュレーションなど、海外では実用的なユースケースが先行している。日本のPLATEAUはオープンデータ化と全国統一基盤という点で独自性があるが、商業化・ビジネスモデル確立では遅れている。2025年度のPLATEAU AWARD開催により、民間企業・スタートアップの参入促進、イノベーション創出を加速する戦略である。

## 日本の立ち位置、弱み・強みの要点（4点）

### 1. 強み：国家主導の統一基盤整備とオープンデータ化
**詳細**: 国土交通省PLATEAUプロジェクトにより、全国約130都市の3D都市モデルがCityGML 2.0国際標準形式で整備され、完全オープンデータとして無償公開されている。これは世界的にも珍しい国家レベルの取り組みであり、データの標準化により異なる都市間でのデータ連携、民間企業の参入障壁低下、イノベーション促進が可能になっている。

**競争優位性**: 
- 中央政府による統一規格と品質管理
- 補助金支援による全国展開の加速
- オープンイノベーションプラットフォームとしての機能
- 自動運転、ドローン運航管理などの先端技術との連携研究

**課題**: 初期整備後のデータ更新体制が脆弱で、補助金依存からの脱却が必要

### 2. 強み：高精度な測量技術と地理空間情報の蓄積
**詳細**: 国際航業、パスコなど国内測量企業は、航空写真測量、LiDAR測量で世界トップクラスの精度を持ち、数cm精度の3D都市モデル作成が可能。ゼンリン、三菱電機のダイナミックマップ基盤（DMP）は、自動運転レベル3以上に対応する高精度3Dマップを提供し、工事・事故などの動的情報をリアルタイム更新する技術を確立している。

**競争優位性**:
- 航空写真測量から3D都市モデル作成まで一貫対応可能
- 自動運転、建設DX、防災などへの多面的展開
- 日本特有の高密度都市、複雑地形への対応力

**課題**: 測量データの商業化・収益化モデルが未成熟、海外展開は限定的

### 3. 弱み：実用的ユースケース不足と費用対効果の説明力
**詳細**: PLATEAUデータは公開されているが、自治体・民間企業による実用的な活用事例が限定的で、「3D都市モデルを作ったが使われていない」という状況が散見される。初期整備コストが数千万円〜数億円と高額な一方、長期的な事故削減効果、渋滞緩和の経済効果、防災減災効果の定量評価が不十分で、自治体の費用対効果理解を得にくい。

**具体的課題**:
- ハッカソン、アイデアソンで生まれたアイデアの事業化率が低い
- 自治体職員のデジタルツイン活用スキル不足
- 3D都市モデルの精度限界（郊外・未整備エリア）
- データ更新の持続性確保（補助金依存）

**対策**: 
- PLATEAU AWARD 2025での優良事例表彰、横展開
- 東京大学など教育機関での人材育成プログラム
- 産官学連携による実証実験の推進

### 4. 弱み：海外企業との商業化競争での遅れ
**詳細**: シンガポールのVirtual Singapore（Dassault Systèmes連携）、韓国のMORAI自動運転シミュレーションなど、海外では政府と民間企業が連携し、実用的なビジネスモデルを確立している。日本のPLATEAUはオープンデータ化を重視するあまり、商業化・収益化の視点が弱く、国内IT企業・スタートアップの競争力向上につながっていない。

**具体的課題**:

## 品質保証エンジニアの視点

### データ品質保証の重要性
**3D都市モデルの精度検証**:
- 航空写真測量データの精度評価（水平精度±数cm、垂直精度±10cm）
- CityGMLデータの属性情報の完全性・整合性検証
- 地上実測データとの照合による精度確認

**シミュレーション妥当性確認**:
- 交通流シミュレーションの実測データとの比較検証
- 事故リスク評価モデルの統計的妥当性確認
- 環境影響評価（騒音、大気質）のセンサーデータとの整合性

**Pythonによるデータ品質評価実装例**:
```python
import geopandas as gpd
import pandas as pd
import numpy as np
from scipy.stats import pearsonr

# CityGMLデータの読み込みと属性情報の完全性チェック
def validate_citygml_attributes(citygml_file):
    """
    3D都市モデルの属性情報の完全性を検証
    """
    gdf = gpd.read_file(citygml_file)
    
    # 必須属性の存在確認
    required_attributes = ['building_id', 'usage', 'height', 'floors']
    missing_attrs = [attr for attr in required_attributes 
                     if attr not in gdf.columns]
    
    # 欠損値の統計
    missing_rate = gdf[required_attributes].isnull().sum() / len(gdf)
    
    return {
        'total_buildings': len(gdf),
        'missing_attributes': missing_attrs,
        'missing_rate': missing_rate.to_dict()
    }

# 交通流シミュレーションの妥当性検証
def validate_traffic_simulation(simulated_data, observed_data):
    """
    シミュレーション結果と実測データの相関分析
    """
    # データの時系列整合
    merged = pd.merge(simulated_data, observed_data, 
                      on='timestamp', suffixes=('_sim', '_obs'))
    
    # 相関係数の計算
    correlation, p_value = pearsonr(
        merged['traffic_volume_sim'], 
        merged['traffic_volume_obs']
    )
    
    # 誤差率の計算
    error_rate = np.abs(
        merged['traffic_volume_sim'] - merged['traffic_volume_obs']
    ) / merged['traffic_volume_obs'] * 100
    
    return {
        'correlation': correlation,
        'p_value': p_value,
        'mean_error_rate': error_rate.mean(),
        'max_error_rate': error_rate.max()
    }

# 事故リスク評価モデルのROC曲線分析
from sklearn.metrics import roc_auc_score, roc_curve
import matplotlib.pyplot as plt

def evaluate_accident_risk_model(risk_scores, actual_accidents):
    """
    事故発生リスクモデルの予測精度評価
    """
    # ROC-AUCスコアの計算
    auc_score = roc_auc_score(actual_accidents, risk_scores)
    
    # ROC曲線のプロット
    fpr, tpr, thresholds = roc_curve(actual_accidents, risk_scores)
    
    plt.figure(figsize=(10, 6))
    plt.plot(fpr, tpr, label=f'ROC curve (AUC = {auc_score:.3f})')
    plt.plot([0, 1], [0, 1], 'k--', label='Random')
    plt.xlabel('False Positive Rate')
    plt.ylabel('True Positive Rate')
    plt.title('Accident Risk Model - ROC Curve')
    plt.legend()
    plt.grid(True)
    plt.savefig('accident_risk_roc.png', dpi=300, bbox_inches='tight')
    
    return {
        'auc_score': auc_score,
        'fpr': fpr,
        'tpr': tpr,
        'thresholds': thresholds
    }

# 実行例
```
