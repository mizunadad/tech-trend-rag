# T9-06-03 自動飛行制御・衝突回避システム

## Summary（5つの要点）

1. **技術概要**: GPSと各種センサーを統合し、AIにより自律飛行と障害物（ビル、他機、鳥等）の自動回避を実現するシステム。パイロット不足解消、運航コスト削減、24時間運航実現に不可欠な技術基盤

2. **自動化レベル**: 航空機自動化はSAE自動運転レベルに相当する段階的実装。レベル3（条件付き自動化、パイロット監視必要）→レベル4（高度自動化、特定条件下で完全自動）→レベル5（完全自動化、無人運航）。2025-2030年はレベル3-4を想定

3. **センサー統合**: GPS/GNSS（全球測位）、IMU（慣性計測装置）、気圧高度計、LiDAR（3D測距）、カメラ（物体認識）、レーダー（他機検知）、ADS-B（航空機自動位置通報）を統合。センサーフュージョンにより冗長性と高精度を確保

4. **AI制御技術**: 深層強化学習（Deep Reinforcement Learning）で最適飛行経路を学習。シミュレーター内で100万回以上の飛行訓練により、緊急時対応（エンジン故障、突風、バードストライク）を習得。認証ハードルは自動運転車より遥かに高い

5. **認証と社会受容性**: 有人機の自動化はレベル4（リモート監視付き無人運航）が2030年代の現実的目標。認証当局（FAA、EASA）がAI制御の安全性証明方法を策定中。説明可能AI（XAI: Explainable AI）による意思決定透明性確保が鍵

## 技術解説詳細

### ハイプ・サイクル上の位置づけ

**現在位置**: 幻滅期の谷から啓蒙活動期への移行段階

- **黎明期（2010-2016）**: DJI等の民生ドローンがGPS自動飛行を普及、有人機への応用構想
- **過度な期待のピーク（2017-2019）**: 「2020年に完全自動運航eVTOL実用化」という楽観的予測
- **幻滅期の谷（2020-2023）**: AI制御の認証ハードル、説明可能性問題、安全性証明の困難さが明確化
- **啓蒙活動期（2024-2028）**: ドローン配送でレベル4自動飛行実証、有人機はレベル3から段階導入
- **生産性の安定期（2029-2035）**: 限定空域・条件下でレベル4無人運航、徐々に適用範囲拡大

### 自動飛行の技術階層

**レイヤー1: センサーデータ取得**
- **GPS/GNSS**: 位置精度1-5m（通常）、RTK-GPS使用で数cm精度達成可能
- **IMU（慣性計測装置）**: 加速度、角速度を計測、姿勢推定に使用
- **気圧高度計**: 高度測定、GPS補完
- **LiDAR**: 3D点群データで障害物検知、測距精度±2cm
- **カメラ**: RGB画像で物体認識（ビル、他機、鳥、ドローン等）
- **レーダー**: 他機検知、悪天候でも動作
- **ADS-B受信機**: 他の航空機の位置・高度・速度を受信

**レイヤー2: 認識・予測**
- **物体検出**: YOLO、Faster R-CNNで画像から障害物を検出
- **セマンティックセグメンテーション**: 画像を「空」「ビル」「地面」「他機」等に分類
- **トラッキング**: 検出した物体の軌跡予測（カルマンフィルター、パーティクルフィルター）
- **衝突リスク評価**: Time to Collision（TTC）計算、危険度スコアリング

**レイヤー3: 経路計画（Path Planning）**
- **グローバル経路計画**: 出発地→目的地の最適経路（A*、Dijkstra、RRT*等のアルゴリズム）
- **ローカル経路計画**: 障害物回避のため、リアルタイムで経路修正（Dynamic Window Approach、APF等）
- **4次元経路計画**: 位置（x,y,z）＋時間を考慮し、他機との空中交通管制を統合

**レイヤー4: 制御（Flight Control）**
- **姿勢制御**: PID制御、LQR（線形二次レギュレータ）で姿勢安定化
- **推力配分**: 複数モーターへの推力指令値生成
- **飛行モード管理**: 離陸→巡航→着陸の各フェーズで制御パラメータ切替
- **緊急対応**: 故障検知時の自動セーフモード移行、緊急着陸地点探索

### AIモデルの認証課題

**問題1: ブラックボックス性**
- **課題**: ニューラルネットワークの意思決定プロセスが不透明
- **解決策**: 説明可能AI（XAI）技術の適用
  - Grad-CAM: 画像のどの領域に注目したかを可視化
  - SHAP: 入力特徴量の寄与度を定量化
  - アテンション機構の可視化

**問題2: コーナーケースへの対応**
- **課題**: 訓練データに含まれない稀な状況（例: 火山灰、隕石落下）への対応不明
- **解決策**:
  - シミュレーションで極端なシナリオを網羅的にテスト
  - Out-of-Distribution（OOD）検知で未知状況を判定し、保守的動作に移行
  - 人間オペレーター介入（レベル3）でリスク低減

**問題3: 敵対的攻撃（Adversarial Attack）**
- **課題**: 微小なノイズ追加でAIの誤認識を誘発可能（例: 「止まれ」標識を「進め」と誤認識）
- **解決策**:
  - Adversarial Training（敵対的サンプルを訓練に含める）
  - アンサンブル学習（複数モデルの多数決）
  - 物理的攻撃耐性（カメラレンズへのレーザー照射対策等）

### 品質保証・信頼性の視点

**DO-178C認証（航空機ソフトウェア）**
- **Level A（壊滅的）**: 故障が墜落につながるソフトウェア→最も厳格
  - 自動飛行制御の主要部分はLevel A相当
  - 要求: 100%のコードカバレッジ、MC/DC（Modified Condition/Decision Coverage）
- **Level B（有害）**: 故障が大怪我につながる
- **Level C（大規模）**: 故障が不便をもたらす程度

**検証手法**
- **MIL（Model-In-the-Loop）**: Simulink等でモデル検証
- **SIL（Software-In-the-Loop）**: 自動生成コードを検証
- **PIL（Processor-In-the-Loop）**: 実機プロセッサで実行検証
- **HIL（Hardware-In-the-Loop）**: 実機ハードウェア統合検証
- **実機飛行試験**: 数千時間の飛行試験でエッジケース確認

**メカトロニクス品質保証への示唆**
- 機械学習モデルのバージョン管理（MLOps）
- データセットの品質管理（アノテーション精度、バイアス除去）
- モデルのドリフト検知（本番環境で性能劣化を監視）
- A/Bテスト（新旧モデルの並行運用で性能比較）

### データ解析・Python活用領域

1. **飛行データ解析**: Pandas、NumPyで飛行ログ解析、異常検知
2. **強化学習訓練**: OpenAI Gym、Stable Baselines3でシミュレーション環境構築
3. **センサーフュージョン**: カルマンフィルター実装、センサー誤差補正

```python
# 例: 簡易衝突回避アルゴリズム（Artificial Potential Field法）
import numpy as np
import matplotlib.pyplot as plt

def attractive_force(pos, goal, k_att=1.0):
    """ゴールへの引力"""
    return k_att * (goal - pos)

def repulsive_force(pos, obstacle, k_rep=100.0, d0=5.0):
    """障害物からの斥力"""
    d = np.linalg.norm(pos - obstacle)
    if d > d0:
        return np.zeros(2)
    return k_rep * (1/d - 1/d0) * (1/d**2) * (pos - obstacle) / d

# シミュレーション
pos = np.array([0.0, 0.0])
goal = np.array([10.0, 10.0])
obstacles = [np.array([5.0, 5.0]), np.array([7.0, 3.0])]
trajectory = [pos.copy()]

for _ in range(100):
    f_att = attractive_force(pos, goal)
    f_rep = sum([repulsive_force(pos, obs) for obs in obstacles])
    pos += 0.1 * (f_att + f_rep)
    trajectory.append(pos.copy())
    if np.linalg.norm(pos - goal) < 0.5:
        break

trajectory = np.array(trajectory)
plt.plot(trajectory[:, 0], trajectory[:, 1], 'b-')
plt.plot(goal[0], goal[1], 'g*', markersize=15)
for obs in obstacles:
    plt.plot(obs[0], obs[1], 'ro', markersize=10)
plt.show()
```

## 実装上の課題と解決策

### 課題1: 認証取得の困難性

**問題**: AIの意思決定プロセスが不透明で、DO-178C認証に適合困難
**解決策**:
- ハイブリッドアプローチ（ルールベース＋AI）で説明可能性向上
- 形式手法（Formal Verification）でAIの振る舞い範囲を数学的に証明
- Safety Case作成（事故シナリオ網羅的分析）

### 課題2: エッジケースへの対応

**問題**: 訓練データに無い稀な状況（バードストライク、ドローンとの接近等）への対応
**解決策**:
- シミュレーションで100万通り以上のシナリオ生成
- OOD検知で未知状況を判定→保守的動作（高度下げ、速度落とす）
- リモート監視オペレーター介入機能（レベル3運用）

### 課題3: サイバーセキュリティ

**問題**: GPS偽装、センサーハッキング、通信妨害のリスク
**解決策**:
- センサー冗長化（GPS＋IMU＋ビジュアルSLAM）で偽装検知
- 暗号化通信（TLS 1.3、VPN）
- 侵入検知システム（IDS）で異常通信を検出
